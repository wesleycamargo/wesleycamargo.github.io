Terminology

  1. Container - is a package with everything necessary to run an application. the smaller unit in the AKS environment,
  2. Pod — is an instance of an application. It can have one or more containers
  3. Deployment — it has one or more identical pods
  4. Nodes— VMs running containerized applications
  5. Pool — Groups of nodes with the same configuration

AKS Cluster architecture

AKS Clusters

Nodes are VMs running containerized applications, and are divided into two types in AKS:

  • Azure-managed nodes — This is a managed node abstracted by Azure, providing the core Kubernetes services and orchestration of applications. 
  • Customer-managed nodes — These clusters are where the customer applications run, being administered by him.
